None of that is required anymore. What follows is the complete process — every prompt, every command, every screen — to go from an idea to a live App Store app using Claude Code as your engineer.

0Setup (10 minutes, one time)

  • Install Claude Code: npm install -g @anthropic-ai/claude-code (needs Node.js 18+)
  • Create a free Expo account at expo.dev, and install the Expo Go app on your iPhone
  • An Apple Developer account ($99/yr) — only needed at Step 5, so you can start today without it

1Let Claude Code set up the project

Open a terminal in an empty folder, run claude, and paste:

Prompt Set up a new React Native app using Expo for an iPhone + Android app called [YOUR APP NAME]. Use TypeScript. Confirm it runs with npx expo start before we continue.

Why this stack: one code base, both platforms. Claude Code will install everything, create the project, and fix its own errors as it goes. Just watch it work.

2Build the simplest version first

Prompt Build the simplest possible version of this idea: [DESCRIBE YOUR APP IN 2-3 SENTENCES]. One screen only. No login, no settings, no extras. I want to see it working today.

Then add features one at a time:

Prompt (repeat per feature) Add one feature: [FEATURE]. Keep everything else unchanged. Show me what to test when done.
Rule that saves weeks: never ask for two features in one prompt. One feature → test on your phone → next feature. This is how you avoid the "everything is broken" spiral.

3Test live on your phone

npx expo start

A QR code appears in the terminal. Scan it with your iPhone camera — the app opens inside Expo Go, live. Every change Claude makes reloads on your phone in seconds.

4Claude writes your App Store listing

Prompt Write my App Store listing: an app name under 30 characters, a subtitle under 30 characters, a description that sells the outcome (not the features), 100 characters of keywords, and what I should show in 5 screenshots.
Screenshots: take them inside the iOS Simulator (Claude will tell you the exact sizes Apple wants: 6.7" and 5.5" displays), or ask Claude to generate a screenshot script.

5Submit for review

Prompt Prepare this app for App Store submission with EAS. Walk me through eas build and eas submit step by step, including certificates — assume I have never done this before.
  • Expo's EAS service handles the scary parts (certificates, signing, upload)
  • Fill the App Store Connect form with the listing from Step 4
  • Review typically takes 24–48 hours. When Apple approves — your app is live, worldwide.

The 5 mistakes that stall beginners

  • Building too much before testing — Step 2's one-feature rule exists for a reason
  • Vague prompts — "make it better" fails; "make the button full-width and blue" ships
  • Skipping the phone test — the simulator lies about feel; Expo Go doesn't
  • Fighting errors yourself — paste the full error back into Claude Code; it fixes its own mess
  • Waiting for the perfect idea — your first app is for learning the loop, not for glory
